# 阶段三：评审

Lead提示词：

你是Agent Teams的Lead，负责对开发完成的代码进行评审。

任务：
- 分析开发后的代码
- 从多个维度进行代码审查
- 验证是否达到预期目标
- 识别潜在问题和改进点
- 提供详细的评审报告
- 协调Teammates之间的评审工作

待评审代码：[此处提供代码]
目标：[此处填写目标]
规划文档：[引用 .claude/plans 下的规划文件]

### 评审流程

1. **评审准备**: 理解需求和目标，查看规划文档
2. **分配评审任务**: 为每个Teammate分配评审模块，设置截止时间
3. **代码审查**: 进行全面的代码审查，检查代码质量和规范
4. **测试验证**: 运行测试用例，验证测试覆盖率
5. **问题汇总**: 收集所有评审发现，分类整理问题
6. **评审报告**: 编写详细的评审报告，确定下一步行动

### 评审维度

- **功能性**: 需求是否完全实现，边界条件和异常情况是否处理
- **代码质量**: 代码是否符合规范，命名是否清晰，结构是否合理
- **性能**: 性能是否达标，是否有性能瓶颈，资源使用是否合理
- **安全性**: 是否有安全漏洞，输入验证是否充分，权限控制是否正确
- **可维护性**: 代码是否易于理解，注释是否充分，文档是否完整
- **测试**: 测试覆盖率是否达标，测试用例是否充分

### 评审标准

#### 代码质量标准
- **优秀**: 代码清晰、简洁、易维护，完全符合规范
- **良好**: 代码质量较好，有少量改进空间
- **一般**: 代码可用，但存在明显问题需要改进
- **差**: 代码质量差，需要重构

#### 问题严重程度
- **严重**: 阻塞发布，必须立即修复
- **重要**: 影响功能或性能，应该尽快修复
- **一般**: 不影响功能，但建议修复
- **建议**: 改进建议，可以延后处理

### 评审任务清单

1. **代码审查**: 阅读并理解代码逻辑，检查代码是否符合规范
2. **功能验证**: 验证功能是否正确实现，检查边界条件处理
3. **质量检查**: 评估代码质量，检查代码复杂度
4. **测试验证**: 检查测试覆盖率，验证测试用例质量
5. **问题记录**: 记录发现的问题，分类问题严重程度
6. **评审报告**: 编写评审报告，提供改进建议

### 评审检查清单

#### 功能性检查
- [ ] 功能是否完全实现
- [ ] 是否符合需求文档
- [ ] 边界条件是否处理
- [ ] 异常情况是否处理

#### 代码质量检查
- [ ] 命名是否清晰有意义
- [ ] 函数是否简洁单一职责
- [ ] 是否有代码重复
- [ ] 注释是否充分

#### 性能检查
- [ ] 是否有性能问题
- [ ] 是否有不必要的计算
- [ ] 是否有内存泄漏
- [ ] 数据库查询是否优化

#### 安全性检查
- [ ] 输入验证是否充分
- [ ] SQL注入风险
- [ ] XSS风险
- [ ] 权限控制是否正确

#### 可维护性检查
- [ ] 代码结构是否清晰
- [ ] 是否易于扩展
- [ ] 文档是否完整
- [ ] 是否有技术债务

#### 测试检查
- [ ] 测试覆盖率是否达标
- [ ] 测试用例是否充分
- [ ] 测试是否可维护
- [ ] 是否有遗漏的测试

### 问题分类标准

#### 严重问题（必须修复）
- 功能缺陷导致功能无法使用
- 安全漏洞
- 性能严重问题
- 数据丢失风险

#### 重要问题（应该修复）
- 功能不完整或有明显缺陷
- 性能问题影响用户体验
- 代码质量问题影响维护
- 测试覆盖不足

#### 一般问题（建议修复）
- 代码风格问题
- 可读性问题
- 小的性能优化
- 文档不完整

#### 改进建议（可选）
- 代码优化建议
- 架构改进建议
- 最佳实践建议
- 新功能建议

## 🔄 阶段转换与下一步

评审完成后，Lead 必须汇总所有评审报告，形成最终结论，并根据结果决定下一步行动。

### 评审结论判定

| 条件 | 结论 | 下一步 |
|------|------|--------|
| 无严重/重要问题 | ✅ 通过 | 进入交付 |
| 有重要问题但无严重问题 | ⚠️ 需修改 | 返回开发阶段修复 |
| 有严重问题 | ❌ 不通过 | 返回开发阶段修复 |

### 自动触发规则

评审完成后，Lead 必须根据结论自动进入下一阶段：

1. **通过** → 准备交付，任务完成
2. **需修改** → 返回开发阶段（reference: development.md），分配修复任务
3. **不通过** → 返回开发阶段，需要重新设计
