## 一、基本元素定义

本章节定义脚本中的基础数据类型、枚举类型和通用属性，是所有命令的基本构建块。

### 1.1 基础数据类型

| 类型 | 说明 | 示例 |
|------|------|------|
| string | 字符串类型 | `"古堡大门"`、`"你好"` |
| number | 数值类型 | `100`、`0.5`、`10` |
| boolean | 布尔类型 | `true`、`false` |
| array | 数组类型 | `["选项1", "选项2"]` |
| condition | 条件表达式 | `"拥有钥匙 == true"` |

**类型使用规则：**
- 属性值类型必须与定义一致
- 变量名可使用中文
- 所有属性名必须是单个英文单词

### 1.2 枚举类型定义

#### 位置枚举（position）

| 值 | 说明 | 默认使用场景 |
|------|------|------|
| left | 左侧位置 | 主角默认位置 |
| center | 中央位置 | 重要角色站位 |
| right | 右侧位置 | 其他角色默认位置 |

#### 动画枚举（animation）

| 值 | 适用命令 | 说明 |
|------|------|------|
| fadein | show | 淡入动画 |
| fadeout | hide | 淡出动画 |
| slidein | show | 滑入动画 |
| slideout | hide | 滑出动画 |

#### 角色类型枚举（type）

| 值 | 说明 |
|------|------|
| main | 主角 |
| npc | 非玩家角色 |
| narrator | 旁白 |

#### 资源类型枚举（type）

| 值 | 说明 |
|------|------|
| cover | 封面图 |
| background | 背景图 |
| bgm | 背景音乐 |
| avatar | 角色头像 |

#### 任务类型枚举（type）

| 值 | 说明 |
|------|------|
| main | 主线任务 |
| side | 支线任务 |

### 1.3 通用属性

以下属性可在任意指令后添加：

| 属性 | 类型 | 说明 |
|------|------|------|
| when | condition | 条件生效 |
| set | 变量 | 设置变量 |
| jump | 目标 | 跳转到场景或命令块 |

**通用属性执行顺序：**

1. `when`：命令执行前判断条件，条件不满足则跳过该命令
2. 命令主体执行
3. `set`：命令执行后设置变量
4. `jump`：命令执行后跳转（jump命令自身不支持附加jump属性）

**示例：**

```yaml
dialog:
  role: "小燕"
  content: "找到钥匙了！"
  when: "拥有钥匙 == true"    # 条件生效
  set: ["金币", "+=50"]       # 执行后设置变量
  jump: "开启宝箱"             # 执行后跳转命令块
```

### 1.4 列表外层属性规则

列表类型指令（show、choice等），统一属性可在列表外层定义，自动应用到所有列表项：

```yaml
show:
  - role: "小燕"
    position: "left"
  - role: "小明"
    position: "right"
animation: "fadein"              # 外层定义，所有角色共享动画
```

## 二、基本命令定义

本章节定义所有可用命令的语法和参数。

### 2.1 命令速查表

| 指令 | 一行写法格式 | 示例 |
|------|------|------|
| 旁白 | `text: ["内容", delay, color]` | `text: ["夜幕降临", 0.5]` |
| 对话 | `dialog: ["角色", "内容", emotion]` | `dialog: ["小燕", "这里是古堡吗？"]` |
| 显示角色 | `show: ["角色", pos, size, animation]` | `show: ["小燕"]` 或 `show: ["小燕", "left"]` |
| 隐藏角色 | `hide: ["角色", animation]` | `hide: ["小燕", "fadeout"]` |
| 跳转 | `jump: ["目标"]` | `jump: ["大厅"]` |
| 设变量 | `set: ["变量名", value]` | `set: ["拥有钥匙", true]` |
| 选项 | `choice: ["文本", target, when]` | `choice: ["推开大门"]` 或 `choice: ["推开大门", "大厅"]` |
| AI输入 | `input: ["提示", "角色", prompt, success, fail]` | `input: ["你想做什么？", "古堡管家"]` |
| 背景 | `bg: ["名称"]` | `bg: ["古堡夜景"]` |
| 音乐 | `bgm: ["名称", loop, fade]` | `bgm: ["神秘氛围", true, 2]` |
| 存档 | `save: ["存档点", attrs]` | `save: []` 或 `save: ["古堡入口"]` |
| 结局 | `end: ["标题", "续集剧本", "场景名称"]` | `end: ["未完待续"]` 或 `end: ["第二章", "神秘古堡2", "入口"]` |

### 2.2 参数定义表

每个指令的必填和可选参数通过下表明确：

| 指令 | 必填参数 | 可选参数 | 参数顺序 |
|------|------|------|------|
| text | content | delay, color | 内容 → 延迟 → 颜色 |
| dialog | role, content | emotion | 角色 → 内容 → 表情 |
| show | role | position, size, animation | 角色 → 位置(默认right) → 大小 → 动画，支持嵌套 |
| hide | role | animation | 角色 → 动画 |
| bg | name | - | 名称 |
| bgm | name | loop, fade | 名称 → 循环 → 淡入 |
| jump | target | - | 目标 |
| set | name, value | - | 变量名 → 值 |
| choice | text | jump, when | 文本 → 跳转目标 → 条件，支持嵌套 |
| input | placeholder, role | prompt, success, fail, max, context | 提示 → 角色 → 指令 → 成功 → 失败 → 最大 → 上下文 |
| save | - | point, attrs | 存档点 → 存档属性列表 |
| end | title | sequel, scene | 标题 → 续集剧本 → 场景名称 |

**解析器处理逻辑：**

1. 数组参数按位置顺序映射
2. 未提供的可选参数使用默认值
3. 必填参数缺失则报错
4. **列表类型指令：外层属性自动应用到所有列表项**

### 2.3 命令详解

#### 2.3.1 text - 旁白

显示旁白文本。

| 属性 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|------|------|
| content | string | 是 | - | 旁白内容 |
| delay | number | 否 | 0 | 延迟显示秒数 |
| color | string | 否 | #FFFFFF | 文字颜色 |

详细写法：

```yaml
# 最简写法
text:
  content: "夜幕降临。"            # 必填：旁白内容

# 带延迟
text:
  content: "夜幕降临。"            # 必填：旁白内容
  delay: 0.5                      # 可选：延迟秒数，默认0

# 完整属性
text:
  content: "夜幕降临。"            # 必填：旁白内容
  delay: 0.5                      # 可选：延迟秒数，默认0
  color: "#AAAAAA"                # 可选：文字颜色，默认#FFFFFF

# 带通用属性
text:
  content: "你发现了钥匙。"        # 必填：旁白内容
  when: "搜索周围 == true"        # 通用属性：条件生效
  jump: "获得提示"                # 通用属性：跳转命令块
```

一行写法：

```yaml
# 最简写法
text: ["夜幕降临。"]              # 仅内容

# 带延迟
text: ["夜幕降临。", 0.5]         # 内容 + 延迟

# 完整属性
text: ["夜幕降临。", 0.5, "#AAAAAA"]  # 内容 + 延迟 + 颜色
```

#### 2.3.2 dialog - 对话

显示角色对话。

| 属性 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|------|------|
| role | string | 是 | - | 角色名称（中文） |
| content | string | 是 | - | 对话内容 |
| emotion | string | 否 | - | 表情切换 |

详细写法：

```yaml

  role: "小燕"                    # 必填：角色名称

  role: "小燕"                    # 必填：角色名称

# 带通用属性
dialog:
  role: "小燕"                    # 必填：角色名称
  content: "我们进去吧！"          # 必填：对话内容
  when: "拥有钥匙 == true"        # 通用属性：条件生效
  jump: "进入提示"                # 通用属性：跳转命令块
```

```yaml
# 最简写法
```

#### 2.3.3 show - 显示角色

显示角色立绘。

| 属性 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|------|------|
| role | string | 是 | - | 角色名称 |
| position | string | 否 | right | 位置：left、center、right（主角默认left，其他角色默认right） |
| size | number | 否 | 100 | 大小百分比 |
| animation | string | 否 | - | 动画效果：fadein、slidein |

**嵌套结构解析（一行写法）：**

- 外层数组：`[角色列表, 动画]`
- 角色列表：`[[角色1, 角色2, ...]]`
- 单个角色：`["角色名", "位置", 大小]`

详细写法：

```yaml
# 单角色 - 最简写法（位置默认right）
show:
  role: "小燕"                    # 必填：角色名称

# 单角色 - 指定位置
show:
  role: "小燕"                    # 必填：角色名称
  position: "left"                # 可选：位置，默认right

# 单角色 - 完整属性
show:
  role: "小燕"                    # 必填：角色名称
  position: "left"                # 可选：位置，默认right
  size: 100                       # 可选：大小百分比，默认100
  animation: "fadein"             # 可选：动画效果

# 多角色 - 推荐写法
show:
  - role: "小燕"                  # 必填：角色名称
    position: "left"              # 可选：位置
    size: 100                     # 可选：大小
  - role: "小明"                  # 必填：角色名称
    position: "right"             # 可选：位置
    size: 80                      # 可选：大小
animation: "fadein"               # 外层定义，所有角色共享动画

# 带通用属性
show:
  role: "小燕"
  position: "left"
  when: "已进入 == true"          # 通用属性：条件生效
  jump: "显示提示"                # 通用属性：跳转命令块
```

一行写法：

```yaml
# 单角色 - 最简写法
show: ["小燕"]                    # 仅角色名，位置默认right

# 单角色 - 指定位置
show: ["小燕", "left"]            # 角色名 + 位置

# 单角色 - 完整属性
show: ["小燕", "left", 100, "fadein"]  # 角色 + 位置 + 大小 + 动画

# 嵌套写法 - 单角色+动画
show: [[["小燕", "left"]], "fadein"]   # [角色列表, 动画]

# 嵌套写法 - 多角色，无动画
show: [[["小燕", "left"], ["小明", "right"]]]

# 嵌套写法 - 多角色+动画
show: [[["小燕", "left"], ["小明", "right"]], "fadein"]

# 复杂场景推荐详细写法
```

#### 2.3.9 choice - 选项

显示玩家选项。

| 属性 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|------|------|
| text | string | 是 | - | 选项文本 |
| jump | string | 否 | - | 跳转目标：场景名称或命令块名称（见下方说明） |
| when | condition | 否 | - | 显示条件 |
| show | boolean | 否 | true | 是否显示 |
| gray | boolean | 否 | false | 是否置灰 |

**jump属性说明：**

- 跳转场景：`jump: "大厅"` - 跳转到指定场景
- 跳转命令块：`jump: "成功_进入大厅"` - 执行命令块（命令块也有name）
- 逻辑统一：跳转场景和跳转命令块的处理逻辑相同

详细写法：

```yaml
# 单选项 - 最简写法
choice:
  - text: "推开大门"              # 必填：选项文本

# 单选项 - 跳转场景
choice:
  - text: "进入大厅"              # 必填：选项文本
    jump: "大厅"                  # 可选：跳转场景

# 单选项 - 执行命令块
choice:
  - text: "推开大门"              # 必填：选项文本
    jump: "成功_进入大厅"          # 可选：跳转命令块

# 单选项 - 带条件
choice:
  - text: "高级技能"              # 必填：选项文本
    jump: "使用技能"              # 可选：跳转命令块
    when: "等级 >= 10"            # 可选：显示条件

# 多选项 - 完整示例
choice:
  - text: "推开大门"              # 必填：选项文本
    jump: "成功_进入大厅"          # 可选：跳转命令块
  - text: "搜索周围"              # 必填：选项文本
    jump: "获得钥匙"              # 可选：跳转命令块
  - text: "秘密通道"              # 必填：选项文本
    jump: "进入密室"              # 可选：跳转命令块
    show: false                   # 可选：是否显示，默认true
  - text: "高级技能"              # 必填：选项文本
    jump: "使用技能"              # 可选：跳转命令块
    when: "等级 >= 10"            # 可选：显示条件
  - text: "直接进入"              # 必填：选项文本
    jump: "大厅"                  # 可选：跳转场景

# 外层条件（所有选项共享）
choice:
  - text: "选项1"
    jump: "命令1"
  - text: "选项2"
    jump: "命令2"
when: "已进入 == true"            # 外层条件，所有选项共享
```

一行写法：

```yaml
# 单选项 - 最简写法
choice: ["推开大门"]              # 仅文本

# 单选项 - 跳转场景
choice: ["推开大门", "大厅"]      # 文本 + 场景名

# 单选项 - 跳转命令块
choice: ["推开大门", "成功_进入"]  # 文本 + 命令块名

# 单选项 - 带条件
choice: ["推开大门", "成功_进入", "拥有钥匙 == true"]  # 文本 + 目标 + 条件

# 多选项嵌套 - 基础
choice: [["推开大门", "成功_进入"], ["搜索周围", "获得钥匙"]]

# 多选项嵌套 - 混合场景和命令块
choice: [["推开大门", "成功_进入"], ["直接进入", "大厅"]]

# 多选项嵌套 - 带条件
choice: [["推开大门", "成功_进入"], ["高级技能", "使用技能", "等级 >= 10"]]

# 复杂场景推荐详细写法
```

#### 2.3.10 input - AI交互

玩家输入，AI自动处理。

| 属性 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|------|------|
| placeholder | string | 是 | - | 玩家输入框提示语 |
| role | string | 是 | - | AI扮演的角色名（中文） |
| prompt | string | 否 | "理解玩家意图" | 给AI的指令（中文） |
| success | string | 否 | - | 成功执行命令块 |
| fail | string | 否 | - | 失败执行命令块 |
| max | number | 否 | 30 | 最大输入字数 |
| context | boolean | 否 | true | 是否记录上下文 |

**context属性说明：**

- `true`（默认）：后台基于「玩家ID+场景」自动记录/关联上下文，AI可获取历史对话
- `false`：不记录当前轮对话，AI仅处理本轮输入，无历史上下文

详细写法：

```yaml
# 最简写法
input:
  placeholder: "你想做什么？"      # 必填：输入框提示语
  role: "古堡管家"                # 必填：AI角色名

# 带AI指令
input:
  placeholder: "你想做什么？"      # 必填：输入框提示语
  role: "古堡管家"                # 必填：AI角色名
  prompt: "识别玩家行动意图"       # 可选：给AI的指令

# 完整属性
input:
  placeholder: "你想做什么？"      # 必填：输入框提示语
  role: "古堡管家"                # 必填：AI角色名
  prompt: "识别玩家行动意图"       # 可选：给AI的指令
  success: "成功_执行操作"        # 可选：成功执行命令块
  fail: "失败_重说"               # 可选：失败执行命令块
  max: 30                         # 可选：最大输入字数，默认30
  context: true                   # 可选：是否记录上下文，默认true

# 关闭上下文（独立问答场景）
input:
  placeholder: "当前时间是多少？"  # 必填：输入框提示语
  role: "智能助手"                # 必填：AI角色名
  prompt: "回复当前系统时间"       # 可选：给AI的指令
  success: "成功_回复时间"        # 可选：成功执行命令块
  fail: "失败_重说"               # 可选：失败执行命令块
  context: false                  # 可选：关闭上下文

# 带通用属性
input:
  placeholder: "你想做什么？"      # 必填：输入框提示语
  role: "古堡管家"                # 必填：AI角色名
  when: "已进入 == true"          # 通用属性：条件生效
```

一行写法：

```yaml
# 最简写法
input: ["你想做什么？", "古堡管家"]  # 提示 + 角色

# 带AI指令
input: ["你想做什么？", "古堡管家", "识别意图"]  # 提示 + 角色 + 指令

# 完整属性
input: ["你想做什么？", "古堡管家", "识别意图", "成功_执行", "失败_重说"]  # 提示 + 角色 + 指令 + 成功 + 失败

# 带最大字数和上下文
input: ["你想做什么？", "古堡管家", "识别意图", "成功_执行", "失败_重说", 30, true]
```

#### 2.3.11 save - 存档

设置存档点。

| 属性 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|------|------|
| point | string | 否 | 当前场景名 | 存档点标识 |
| attrs | array | 否 | 全部属性 | 存档属性列表 |

**默认存储属性：**

- 当前场景名称
- 当前章节
- 所有游戏变量（attributes.yaml中定义的）
- 玩家选择历史

详细写法：

```yaml
# 最简写法 - 使用默认存档点（当前场景名）
save: {}

# 指定存档点名称
save:
  point: "古堡入口"              # 可选：存档点标识，默认当前场景名

# 自定义存档属性
save:
  point: "古堡入口"              # 可选：存档点标识
  attrs:                         # 可选：存档属性列表，默认全部
    - "金币"
    - "拥有钥匙"
    - "已进入"

# 存档所有属性（默认行为）
save:
  point: "古堡入口"
  attrs: "all"                   # 存档所有属性

# 带通用属性
save:
  point: "古堡入口"
  when: "已进入 == true"         # 通用属性：条件生效
```

一行写法：

```yaml
# 最简写法 - 使用默认存档点
save: []                         # 存档点为当前场景名，存储全部属性

# 指定存档点
save: ["古堡入口"]               # 存档点 + 全部属性

# 自定义存档属性
save: ["古堡入口", ["金币", "拥有钥匙"]]  # 存档点 + 属性列表
```

#### 2.3.12 end - 结局

## 三、组合规则

本章节定义命令组合、场景组合、章节组合的规则和约束条件。

### 3.1 命令组合规则

#### 3.1.1 命令分类

根据命令的功能和执行特性，将所有命令分为以下类别：

| 类别 | 命令 | 说明 |
|------|------|------|
| 展示类 | text, dialog, show, hide | 用于展示内容和角色 |
| 环境类 | bg, bgm | 用于设置场景环境 |
| 流程类 | jump, choice, end | 用于控制剧情流程 |
| 数据类 | set, save | 用于管理游戏数据 |
| 交互类 | input | 用于AI交互 |

#### 3.1.2 命令组合规则

**规则1：通用属性附加规则**

所有命令都可以附加以下通用属性：

| 通用属性 | 适用命令 | 执行时机 |
|----------|----------|----------|
| when | 全部命令 | 命令执行前判断条件 |
| set | 全部命令 | 命令执行后设置变量 |
| jump | 全部命令（除jump自身） | 命令执行后跳转 |

```yaml
# 示例：命令+通用属性组合
dialog:
  role: "小燕"
  content: "找到钥匙了！"
  when: "拥有钥匙 == true"    # 条件生效
  set: ["金币", "+=50"]       # 执行后设置变量
  jump: "开启宝箱"             # 执行后跳转命令块
```

**规则2：命令执行顺序**

在`steps`数组中，命令按数组顺序依次执行：

```yaml
steps:
  - bg: ["古堡夜景"]           # 1. 先设置背景
  - bgm: ["神秘氛围"]          # 2. 再播放音乐
  - show: ["小燕", "left"]     # 3. 显示角色
  - dialog: ["小燕", "你好"]   # 4. 角色对话
  - choice: [...]              # 5. 最后显示选项
```

**规则3：推荐命令组合顺序**

| 阶段 | 推荐命令 | 说明 |
|------|----------|------|
| 场景初始化 | bg → bgm | 先设置环境 |
| 角色入场 | show → dialog | 显示角色后对话 |
| 剧情交互 | text/dialog → choice/input | 叙事后提供选择 |
| 场景切换 | hide → jump | 角色离场后跳转 |
| 场景结束 | save → end | 存档后结局 |

**规则4：互斥命令组合**

以下命令不应在同一`steps`中相邻出现：

| 命令组合 | 问题 | 建议 |
|----------|------|------|
| end → end | 重复结局 | 仅保留一个 |
| jump → jump | 第一个jump失效 | 合并或删除 |
| choice → choice | 选项覆盖 | 分离到不同场景 |
| bg → bg (连续) | 背景闪烁 | 添加过渡命令 |

#### 3.1.3 嵌套深度限制

| 结构 | 最大嵌套深度 | 说明 |
|------|--------------|------|
| choice选项 | 2层 | `[选项列表, 外层条件]` |
| show角色列表 | 2层 | `[[角色列表], 动画]` |
| 命令块steps | 1层 | 命令块内不能再嵌套命令块 |
| 通用属性 | 1层 | 不支持多层when/set/jump嵌套 |

```yaml
# 正确：choice嵌套深度为2
choice: [[["选项1", "目标1"], ["选项2", "目标2"]], "外层条件"]

# 错误：嵌套过深
choice: [[[[...]]]]  # 超过2层，解析错误
```

***

### 3.2 场景组合规则

#### 3.2.1 场景定义规则

**规则名称：场景结构定义**

场景由以下部分组成：

```yaml
scenes:
  - name: "场景名称"          # 必填：场景唯一标识
    bg: ["背景名称"]          # 可选：场景背景
    bgm: ["音乐名称"]         # 可选：场景音乐
    steps:                    # 必填：命令序列
      - ...
```

**约束条件：**
- `name`必须唯一（scenes.yaml内）
- `steps`至少包含一个命令
- `bg`和`bgm`为场景级别属性，优先于steps内的同名命令

#### 3.2.2 场景属性继承规则

**规则名称：场景属性继承**

场景级属性（bg、bgm）与steps内命令的关系：

| 场景属性 | steps内命令 | 执行结果 |
|----------|-------------|----------|
| 有bg | 无bg命令 | 使用场景bg |
| 有bg | 有bg命令 | steps内bg覆盖场景bg |
| 无bg | 有bg命令 | 使用steps内bg |
| 无bg | 无bg命令 | 报错（缺少背景） |

```yaml
# 示例：场景属性与命令覆盖
- name: "古堡大门"
  bg: ["古堡夜景"]          # 场景默认背景
  bgm: ["神秘氛围"]         # 场景默认音乐
  steps:
    - text: ["..."]         # 使用场景默认bg/bgm
    - bg: ["大厅"]          # 覆盖背景
    - text: ["..."]         # 使用新背景
```

**规则名称：属性生命周期**

| 属性 | 生命周期 | 跨场景行为 |
|------|----------|------------|
| bg | 当前场景 | 不继承，新场景需重新定义 |
| bgm | 持续播放 | 继续播放直到新的bgm命令 |
| show的角色 | 持续显示 | 跨场景保持（需手动hide） |

#### 3.2.3 场景与命令序列关系

**规则名称：命令序列执行规则**

```yaml
steps:
  - text: ["..."]      # 步骤1
  - dialog: [...]      # 步骤2
  - choice: [...]      # 步骤3（等待玩家选择）
  # 执行choice后，暂停等待玩家操作
```

**执行流程：**
1. 按数组顺序依次执行
2. 遇到`choice`或`input`时暂停，等待玩家操作
3. 玩家操作后，执行对应的`jump`跳转
4. 遇到`end`时结束当前场景

**阻塞命令说明：**

| 命令 | 是否阻塞 | 后续命令执行条件 |
|------|----------|------------------|
| text | 否 | 立即执行下一个 |
| dialog | 否 | 立即执行下一个 |
| show/hide | 否 | 立即执行下一个 |
| bg/bgm | 否 | 立即执行下一个 |
| set/save | 否 | 立即执行下一个 |
| choice | 是 | 玩家选择后跳转 |
| input | 是 | AI响应后继续 |
| jump | 是 | 立即跳转，不执行后续命令 |
| end | 是 | 结束场景，不执行后续命令 |

***

### 3.3 章节组合规则

#### 3.3.1 章节定义规则

**规则名称：章节结构定义**

```yaml
chapters:
  - name: "章节名称"         # 必填：章节唯一标识
    scenes:                  # 必填：场景名称列表
      - "场景1"
      - "场景2"
```

**约束条件：**
- `name`必须唯一（chapters.yaml内）
- `scenes`列表中的场景名称必须在`scenes.yaml`中存在
- `scenes`列表可以为空（用于占位章节）

#### 3.3.2 章节与场景引用关系

**规则名称：场景引用规则**

```yaml
# chapters.yaml
chapters:
  - name: "第一章"
    scenes:
      - "古堡大门"     # 引用scenes.yaml中的场景
      - "大厅"

# scenes.yaml
scenes:
  - name: "古堡大门"   # 被引用的场景
    steps: [...]
  - name: "大厅"
    steps: [...]
```

**引用完整性约束：**
- 章节引用的场景必须在`scenes.yaml`中定义
- 一个场景可以被多个章节引用
- 场景可以不被任何章节引用（未使用的场景）

#### 3.3.3 章节执行流程

**规则名称：章节流程控制**

```
章节开始
    ↓
第一个场景
    ↓
场景内steps执行
    ↓
遇到jump → 跳转目标场景（可跨章节）
    ↓
遇到end → 章节结束
```

**章节边界规则：**
- 章节边界仅用于组织结构
- `jump`可以跳转到任意章节的场景
- `end`结束整个故事，而非仅当前章节

***

### 3.4 约束条件

#### 3.4.1 必填参数校验

**规则名称：必填参数完整性**

每个命令执行前必须校验必填参数：

| 命令 | 必填参数 | 校验规则 |
|------|----------|----------|
| text | content | 非空字符串 |
| dialog | role, content | role必须在characters.yaml中存在 |
| show | role | role必须在characters.yaml中存在 |
| hide | role | role必须在characters.yaml中存在 |
| bg | name | name必须在resources.yaml中存在（type=background） |
| bgm | name | name必须在resources.yaml中存在（type=bgm） |
| jump | target | target必须在scenes.yaml或commands.yaml中存在 |
| set | name, value | name建议在attributes.yaml中定义 |
| choice | text (每项) | 非空字符串 |
| input | placeholder, role | role必须在characters.yaml中存在 |
| save | - | 无必填参数 |
| end | title | 非空字符串 |

**校验时机：**
- 脚本加载时进行静态校验
- 运行时进行动态校验（条件相关）

```yaml
# 错误示例：缺少必填参数
dialog:
  role: "小燕"
  # 缺少content，解析报错

# 错误示例：角色不存在
dialog: ["未知角色", "你好"]  # "未知角色"未在characters.yaml定义
```

#### 3.4.2 引用完整性校验

**规则名称：跨文件引用完整性**

所有跨文件引用必须验证目标存在：

| 引用方 | 被引用方 | 校验规则 |
|--------|----------|----------|
| chapters.scenes | scenes.name | 必须存在 |
| scenes.bg | resources.name (type=background) | 必须存在 |
| scenes.bgm | resources.name (type=bgm) | 必须存在 |
| dialog.role | characters.name | 必须存在 |
| show.role | characters.name | 必须存在 |
| jump.target | scenes.name 或 commands.name | 必须存在 |
| choice.jump | scenes.name 或 commands.name | 必须存在 |
| input.role | characters.name | 必须存在 |
| input.success | commands.name | 如果提供则必须存在 |
| input.fail | commands.name | 如果提供则必须存在 |
| end.sequel | story.title | 如果提供则必须存在 |
| end.scene | scenes.name | 如果提供则必须存在 |

**错误处理：**
- 引用不存在时，脚本加载失败
- 错误信息需包含：文件名、引用位置、缺失目标

```yaml
# 错误示例：引用不存在的场景
jump: ["不存在的场景"]  # 场景"不存在的场景"未定义
```

#### 3.4.3 循环引用检测规则

**规则名称：循环引用检测**

检测以下循环引用情况：

**1. 场景跳转循环**

```
场景A --jump--> 场景B --jump--> 场景A  # 循环！
```

检测方法：构建场景跳转图，检测是否有环

```yaml
# 循环引用示例
scenes:
  - name: "场景A"
    steps:
      - jump: ["场景B"]
  - name: "场景B"
    steps:
      - jump: ["场景A"]  # 检测到循环！
```

**2. 命令块循环**

```
命令块A --jump--> 命令块B --jump--> 命令块A  # 循环！
```

```yaml
# 命令块循环示例
commands:
  - name: "命令A"
    steps:
      - jump: ["命令B"]
  - name: "命令B"
    steps:
      - jump: ["命令A"]  # 检测到循环！
```

**3. 场景与命令块混合循环**

```
场景A --jump--> 命令块B --jump--> 场景A  # 循环！
```

**循环检测算法：**

```
1. 构建引用图：节点=场景/命令块，边=jump引用
2. 使用深度优先搜索(DFS)检测环
3. 发现环时报告循环路径
```

**允许的循环情况：**

以下情况虽然是循环，但是是合法的设计模式：

```yaml
# 合法：条件跳转形成循环
- name: "探索房间"
  steps:
    - choice:
        - text: "继续探索"
          jump: "探索房间"    # 条件循环，合法
        - text: "离开"
          jump: "其他场景"
```

**循环检测例外规则：**

| 情况 | 是否报错 | 说明 |
|------|----------|------|
| 无条件的直接循环 | 报错 | 死循环 |
| 通过choice的循环 | 警告 | 可能是设计意图 |
| 通过when条件的循环 | 警告 | 条件可能打破循环 |
| jump到自身 | 警告 | 可能是设计意图 |

#### 3.4.4 其他约束条件

**规则名称：场景完整性**

| 约束 | 说明 |
|------|------|
| 每个场景必须有至少一个步骤 | steps不能为空数组 |
| 结局场景必须包含end命令 | 用于标识故事结束点 |
| 入口场景必须存在 | 第一个章节的第一个场景必须定义 |

**规则名称：资源类型匹配**

| 命令 | 资源类型要求 |
|------|--------------|
| bg | resources.type = "background" |
| bgm | resources.type = "bgm" |
| show (角色立绘) | characters.avatar存在 |
| dialog (角色头像) | characters.avatar存在 |

**规则名称：变量类型一致性**

```yaml
# attributes.yaml
attributes:
  - name: "金币"
    type: "number"      # 定义为数值类型

# scenes.yaml - 正确
set: ["金币", 100]       # 数值赋值

# scenes.yaml - 错误
set: ["金币", "一百"]     # 类型不匹配
```

| 变量类型 | 允许的值 |
|----------|----------|
| number | 数字、增量表达式(+=10) |
| boolean | true、false |
| string | 字符串 |

### 3.5 组合规则速查表

#### 3.5.1 命令组合速查

| 组合场景 | 推荐命令序列 |
|----------|--------------|
| 新场景开场 | bg → bgm → text → show → dialog |

## 四、文件结构说明

本章节定义多文件组织方式和文件引用关联。

### 4.1 文件列表

| 文件 | 文件名 | 说明 |
|------|------|------|
| 故事 | story.yaml | 故事标题、描述等 |
| 章节 | chapters.yaml | 所有章节定义 |
| 场景 | scenes.yaml | 所有场景定义 |
| 角色 | characters.yaml | 角色定义 |
| 命令块 | commands.yaml | 可复用命令块定义 |
| 资源 | resources.yaml | 背景图、音乐、立绘 |
| 属性 | attributes.yaml | 属性定义 |
| 任务 | tasks.yaml | 任务定义 |

### 4.2 文件引用关联

各文件之间通过**名称**进行引用关联：

```
story.yaml (故事信息)
    ↓
chapters.yaml (章节列表)
    ↓ 引用场景名称
scenes.yaml (场景定义)
    ├─→ characters.yaml (角色名称)
    ├─→ resources.yaml (资源名称：背景、音乐)
    ├─→ attributes.yaml (通用属性)
    ├─→ commands.yaml (命令块名称)
    └─→ tasks.yaml (任务名称)
characters.yaml (角色定义)
    ├─→ resources.yaml (资源名称：头像、立绘)
    └─→ attributes.yaml (角色属性)
commands.yaml (命令块定义)
resources.yaml (资源定义)
attributes.yaml (属性定义)
tasks.yaml (任务定义)
```

**引用规则：**

| 引用方 | 被引用方 | 引用方式 | 示例 |
|--------|----------|----------|------|
| chapters.yaml | scenes.yaml | 场景名称 | `scenes: ["古堡大门", "大厅"]` |
| scenes.yaml | characters.yaml | 角色名称 | `role: "小燕"` |
| scenes.yaml | resources.yaml | 资源名称 | `bg: ["古堡夜景"]` |
| scenes.yaml | attributes.yaml | 属性名称 | `when: "拥有钥匙 == true"` |
| scenes.yaml | commands.yaml | 命令块名称 | `jump: "成功_进入大厅"` |
| characters.yaml | resources.yaml | 资源名称 | `avatar: "小燕头像"` |
| characters.yaml | attributes.yaml | 属性名称 | 角色初始属性 |
| end指令 | story.yaml | 剧本名称 | `sequel: "神秘古堡第二章"` |
| jump指令 | scenes.yaml | 场景名称 | `jump: "大厅"` |
| jump指令 | commands.yaml | 命令块名称 | `jump: "成功_进入大厅"` |

**名称匹配规则：**
- 所有引用通过`name`属性进行匹配
- 名称区分大小写
- 名称可使用中文
- 名称必须唯一（同一文件内）

### 4.3 基本规则

1. 支持**详细写法**、**一行写法**，顺序一致
2. 不写属性 = 自动用默认值
3. **所有属性名必须是单个英文单词**，内容/命令块名可用中文

### 4.4 快速记忆

- **通用属性**：`when`、`set`、`jump`可在任意指令后添加
- **jump属性**：统一跳转场景或命令块，逻辑一致
- **属性命名**：所有属性名使用英文单词，变量名可用中文
- **条件属性**：统一使用英文`when`
- **列表外层属性**：统一属性在外层定义，自动应用所有列表项
- **show位置**：默认`right`（主角默认left）
- **choice选项**：`jump`统一跳转场景或命令块
- **save存档**：默认存储全部属性，可自定义属性列表
- **end结局**：支持续集跳转，显示续集标题
- **AI交互**：`placeholder`+`role`（必填），其他可选
- **多轮AI**：顺序写input，上下文由`context`控制（默认`true`）

## 五、高级功能

本章节介绍命令块、多轮AI交互和条件表达式等高级功能。

### 5.1 命令块

命令块用于复用一组命令，独立存储在`commands.yaml`文件中，只写中文`name`，系统自动转唯一标识。

#### commands.yaml

```yaml
commands:
  - name: 失败_重说
    steps:
      - text: ["古堡管家", "我没理解你的意思，请换种说法～"]

  - name: 成功_最终执行
    steps:
      - dialog: ["古堡管家", "已为你打开{item}！"]
      - set: ["actiondone", true]

  - name: 获得_钥匙
    steps:
      - text: ["你在角落发现了一把钥匙"]
      - set: ["haskey", true]
```

**命令块属性：**

| 属性 | 类型 | 必填 | 说明 |
|------|------|------|------|
| name | string | 是 | 命令块名称（中文，用于引用） |
| steps | array | 是 | 命令步骤列表 |

***

### 5.2 多轮AI交互

#### 场景1：默认自动记录上下文（最常用）

省略`context`，默认`true`，自动记录并关联历史对话。

```yaml
# 第一轮AI（自动记录上下文）
input:
  placeholder: "你想做什么？"
  role: "古堡管家"
  prompt: "识别玩家行动意图"
  success: ""
  fail: "失败_重说"

# 第二轮AI（自动关联第一轮上下文）
input:
  placeholder: "你具体想操作哪个物品？"
  role: "古堡管家"
  prompt: "提取物品名（参考历史意图）"
  success: ""
  fail: "失败_重说"

# 第三轮AI（获取前两轮上下文）
input:
  placeholder: "确认/取消？"
  role: "古堡管家"
  prompt: "判断是否执行（结合历史意图+物品名）"
  success: "成功_最终执行"
  fail: "失败_重说"
```

#### 场景2：手动关闭上下文

独立问答场景，无需历史上下文。

```yaml
input:
  placeholder: "当前时间是多少？"
  role: "智能助手"
  prompt: "回复当前系统时间"
  success: "成功_回复时间"
  fail: "失败_重说"
  context: false  # 关闭上下文
```

***

### 5.3 条件表达式

#### when条件运算符

| 运算符 | 说明 | 示例 |
|------|------|------|
| == | 等于 | `拥有钥匙 == true` |
| != | 不等于 | `金币 != 0` |
| > | 大于 | `等级 > 10` |
| < | 小于 | `生命值 < 50` |
| >= | 大于等于 | `勇气 >= 5` |
| <= | 小于等于 | `金币 <= 100` |
| && | 与 | `拥有钥匙 && 金币 >= 100` |
| \|\| | 或 | `等级 >= 10 \|\| 会员 == true` |

#### 示例

```yaml
set:
  name: "奖励"
  value: 100
  when: "会员 == true"

jump:
  target: "密室"
  when: "拥有钥匙 && 金币 >= 50"

choice:
  - text: "高级技能"
    jump: "使用技能"
    when: "等级 >= 10 && 法力 >= 20"
```

```yaml
            jump: "逃跑结局"

    bg: ["大厅"]

    bg: ["古堡夜景"]
```
