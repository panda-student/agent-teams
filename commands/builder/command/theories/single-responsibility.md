# 单一职责原则

## 1. 核心定义
单一职责原则（Single Responsibility Principle, SRP）要求每个文件、模块或组件只负责一个明确的功能领域，避免职责混杂，提高可维护性和可读性。

## 2. 在 Skill 设计中的应用

### 文件职责划分
- **SKILL.md**：智能路由器，只负责路由和核心提示
- **reference.md**：详细规范，只负责规范说明
- **examples.md**：示例文档，只负责示例展示
- **theories/**：理论文件，只负责理论阐述
- **templates/**：模板文件，只负责模板定义

### 理论文件职责
- **progressive-disclosure.md**：只阐述渐进式披露理论
- **task-skill-theory.md**：只阐述任务型 Skill 理论
- **task-skill-checklist.md**：只提供七步设计清单
- **single-responsibility.md**：只阐述单一职责原则

## 3. 判断标准

### 单一职责 ✅
- 文件名准确反映内容
- 文件内容聚焦单一主题
- 修改原因唯一

### 违反单一职责 ❌
- 文件名过于宽泛
- 文件内容涉及多个主题
- 修改原因多样

## 4. 实践示例

### ❌ 违反单一职责
```markdown
# task-skill-theory.md
包含：
- 任务型 Skill 理论
- 七步设计清单
- 示例代码
```

### ✅ 符合单一职责
```markdown
# task-skill-theory.md
只包含：
- 任务型 Skill 理论

# task-skill-checklist.md
只包含：
- 七步设计清单
```

## 5. 优势

### 可维护性
- 修改某个功能时，只需修改对应文件
- 避免牵一发动全身

### 可读性
- 文件职责清晰，易于理解
- 快速定位所需内容

### 可复用性
- 独立职责便于复用
- 减少重复代码

## 6. 与其他原则的关系

### 与渐进式披露的关系
- 单一职责支持渐进式加载
- 按需加载独立职责的文件

### 与最小权限原则的关系
- 文件职责单一，权限配置更精确
- 避免过度授权

## 7. 最佳实践

### 文件拆分
- 当文件涉及多个主题时，拆分为独立文件
- 每个文件聚焦单一主题

### 命名规范
- 文件名准确反映职责
- 避免过于宽泛的命名

### 定期审查
- 定期检查文件职责是否单一
- 及时重构违反单一职责的文件
