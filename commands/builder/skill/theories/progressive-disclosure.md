# 渐进式披露架构核心理论

## 1. 核心定义
渐进式披露（Progressive Disclosure）是 Claude Code Skills 体系的核心架构模式，指**分层、按需加载知识与资源**，避免一次性载入全部内容，解决 Token 爆炸与信息噪声问题，本质是 AI 架构下的企业知识管理工程化实现。

## 2. 三层认知架构
- **目录层（Entry Point）**：仅加载 Skill 的 `description`，用于全局语义匹配与触发判断；
- **路由层（Main Content）**：激活后加载 SKILL.md 主文件，承担任务路由与流程指引功能；
- **附录层（On-Demand Resources）**：仅在明确需要时加载参考文档、模板、脚本、数据等附属资源。

## 3. 认知经济学基础
上下文窗口是 LLM 稀缺的工作记忆资源，存在注意力稀释效应；渐进式披露追求**知识 ROI 最大化**，以最小 Token 投入实现最高任务完成质量。

## 4. 路由与契约理论
- SKILL.md 定位为**智能路由器**，通过高信息密度的路由表实现快速分发；
- 采用**契约式引用**，明确触发条件、文件路径、内容预期，保证 AI 对资源加载的可预测性。

## 5. 内容拆分原则
- 高频核心语义内联于主文件，低频细节、示例、数据外置；
- SKILL.md 遵循 **500 行上限原则**，超出则需重构拆分；
- 确定性逻辑封装为脚本，实现"知识预编译"，降低推理与上下文开销。

## 6. Skills × Tools 共生理论
- **Skills 约束 Tools**：通过权限配置实现最小权限与安全边界；
- **Skills 编排 Tools**：用脚本固化工具调用序列，简化 AI 推理链；
- **Tools 反哺 Skills**：通过前置命令注入实时数据，增强知识时效性。

## 7. 上下文管理互补理论
渐进式披露（Skill 内按需加载）与子代理隔离（多任务上下文纯净分割）构成互补策略，共同解决 LLM 上下文效率与干扰问题。
