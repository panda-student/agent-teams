# Agent Teams

多 Agent 协作技能，用于处理各类软件开发任务。

## 触发场景

当用户表达以下意图时自动触发：

| 场景 | 关键词示例 |
|------|-----------|
| **开发模式** | 开发、实现、新增、重构、添加功能 |
| **修复Bug模式** | bug、修复、问题、错误、缺陷、异常、无法、失败 |
| **评审检查模式** | 评审、检查、审查、审计、代码质量 |

## 安装

```bash
# 克隆仓库
git clone git@github.com:panda-student/agent-teams.git

# 复制技能到本地
cp -r agent-teams/skills/agent-teams ~/.openclaw/workspace/skills/
```

## 工作流程

```
用户请求
    │
    ▼
┌─────────────────────────────────────────┐
│           需求分析                        │
│  需求明确 → 自动判断模式                  │
│  需求不明确 → 让用户选择                  │
└───────────────────┬─────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────┐
│           规划-审批阶段                   │
│  输出计划文档 → 等待用户确认              │
└───────────────────┬─────────────────────┘
                    │
                    ▼
┌───────────┬───────────────┬─────────────┐
│  开发模式  │  修复Bug模式   │ 评审检查模式 │
└───────────┴───────────────┴─────────────┘
                    │
                    ▼
              输出结果
```

## 执行模式

| 模式 | 适用场景 | 文档输出位置 |
|------|----------|-------------|
| **开发模式** | 新功能开发、需求实现、代码重构 | `.claude/plans/` |
| **修复Bug模式** | 问题排查、bug修复、缺陷处理 | `.claude/bugs/` |
| **评审检查模式** | 代码评审、质量检查、安全审计 | `.claude/reviews/` |
